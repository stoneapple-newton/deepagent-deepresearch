# Research Findings: OpenCode Effective Use

## Key Facts Synthesized

### What is OpenCode?
- Open-source AI coding agent (160K+ GitHub stars, 900+ contributors, 7.5M+ monthly users)
- Available as: Terminal TUI (primary), Desktop app (beta), IDE extension
- Lives at github.com/anomalyco/opencode (moved from opencode-ai/opencode Sep 2025)
- Go-based, built with Bubble Tea framework

### Installation Options
1. `curl -fsSL https://opencode.ai/install | bash` (recommended)
2. `npm install -g opencode-ai` (Node.js)
3. `brew install anomalyco/tap/opencode` (Homebrew)
4. `docker run -it --rm ghcr.io/anomalyco/opencode` (Docker)
5. Desktop app at opencode.ai/download

### Provider Support (75+)
- OpenCode Zen (curated, tested models), OpenCode Go (low-cost)
- Anthropic Claude, OpenAI GPT, Google Gemini, GitHub Copilot
- Local: Ollama, LM Studio, llama.cpp
- OpenRouter, DeepSeek, xAI Grok, Z.AI GLM, and many more
- Note: Anthropic blocked OpenCode from using Claude models Jan 2026 (plugins removed in v1.3.0)

### Core Workflow: Plan → Build
- **Plan agent**: Read-only analysis (edit: deny, bash: deny/ask). Use for designing approach.
- **Build agent**: Full tool access. Use for implementation.
- Toggle with Tab key. Build-switch system prompt reminds agent it can now make changes.

### AGENTS.md
- Project-specific instructions for AI agents (also used by Cursor, Windsurf, Claude Code, etc.)
- Generated by `/init` command which scans project files
- Should be committed to Git
- Over 60K open-source projects use this format
- Stewarded by the Agentic AI Foundation under the Linux Foundation

### Agent System
- **Built-in primary agents**: Build (default), Plan
- **Built-in subagents**: General (full tool access), Explore (read-only codebase exploration), Scout (external dependency research, read-only)
- **Custom agents**: Defined in opencode.json (JSON) or .opencode/agents/*.md (markdown)
- **Modes**: primary (Tab cycle), subagent (@mention only), all (both)
- **Orchestration**: Primary agent uses task tool to spawn subagents with their own context

### Permissions (allow / ask / deny)
- Keys: read, edit, glob, grep, list, bash, task, external_directory, webfetch, websearch, lsp, skill, todowrite
- Granular control: glob patterns for bash commands (`"git status *": "allow"`)
- Glob file edit permissions for workspace path control
- Permission inheritance: agent overrides global

### MCP (Model Context Protocol)
- Integrates external tools/services (database access, web search, browser automation)
- Configured in opencode.json
- Auto-starts MCP servers on demand

### Key Commands
- `/init` - Generate AGENTS.md
- `/connect` - Add provider
- `/models` - Browse/select models
- `/undo`, `/redo` - Edit history
- `/share` - Create public share link
- `/compact` - Summarize session to free context
- `/export` - Export to markdown
- `/themes` - Change theme

### Keyboard Shortcuts
- Tab - Switch primary agents
- Ctrl+X n - New session
- Ctrl+X l - List sessions
- Ctrl+X c - Compact session
- Ctrl+X u/r - Undo/Redo
- Ctrl+P - Command palette
- Ctrl+O - Model selection
- Ctrl+L - View logs
- Ctrl+A - Switch session
- Esc - Close overlay/return
- Ctrl+E - Open external editor

### Non-Interactive Mode
- `opencode run "message"` - Run non-interactively
- `opencode serve` - Headless server
- `opencode run --attach http://localhost:4096 "prompt"` - Attach to server
- `--quiet` / `-q` flag for script usage

### GitHub Workflow
- `/opencode explain this issue` - Explain issues
- `/opencode fix this` - Auto-create branch + PR
- Comment on PRs to request changes

### Community Power-User Patterns
- Orchestrator + specialist subagents with different models
- Custom slash commands (commit, PR creation, code review, etc.)
- MCP server integration (Anytype, Mem0, browser automation, security scanning)
- Permission sandboxing per agent
- Task permissions to control which agents can invoke which subagents
- Background agents running in parallel (v1.14+)
